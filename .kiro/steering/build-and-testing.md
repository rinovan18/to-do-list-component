# Build & Testing

## Build Commands

Run from an individual component directory (`elements/my-element/`):

```bash
yarn run build     # full build: formats code, generates custom-elements.json
yarn run dev       # build with file watching
```

Run from the monorepo root (`webcomponents/`):

```bash
yarn run build     # builds all components via Lerna
```

**Always run `yarn run build` after editing a HAXCMSLitElement theme.** The build pipeline runs:
1. Gulp — asset compilation
2. Prettier — code formatting
3. CEM — generates `custom-elements.json` (do NOT edit this file manually)

## Testing

```bash
npm test                              # run tests from component or project root
npm run test --filter <component>     # test a specific component in the monorepo
```

## Linting

```bash
npm run lint          # check code style (ESLint + Prettier + Stylelint)
npm run lint:fix      # auto-fix issues
```

## DDD Compliance Audit

Run from the component root before any PR:

```bash
hax audit
```

Checks CSS and structure against DDD design system rules. Files in `.dddignore` are skipped.

### Suggested `.dddignore` contents

```
/node_modules
/dist
/build
/public
/test
/.github
/.vscode
/.idea
/locales
LICENSE
.dddignore
.editorconfig
.gitignore
.npmignore
*.md
*.yml
*.json
*.png
*.svg
```

## Pre-PR Checklist

1. `npm test` — all tests pass
2. `npm run lint` — no lint errors
3. `hax audit` — DDD compliance confirmed (for theme/component work)
4. `yarn run build` — `custom-elements.json` is up to date

## Lerna (Monorepo Management)

```bash
lerna run build              # run build across all packages
lerna add <pkg> --scope=@haxtheweb/<component>  # add dependency to one component
lerna publish                # publish (maintainers only)
lerna clean && lerna bootstrap  # reset if Lerna gets into a bad state
```
